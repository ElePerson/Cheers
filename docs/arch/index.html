关键运行流

下面是已经进入代码的核心契约：写后投递、事件时钟、任务链和资源一致性。

1 消息写入 验证成员、解析 token mention、事务分配 channel_seq、插入消息。
2 终态 fan-out PG commit 后再广播浏览器终态帧，满足 Write-Before-Deliver。
3 mesh dispatch 有 @bot 派发指定 bot；否则使用 default_bot_id；否则静默。
4 bot finalize placeholder finalize 时取 channel_seq，规范化 bot mention，并可触发下一跳。
5 资源追赶 bot 通过 since_seq、by-seq 和 activity stream 补齐上下文。

外部 agent 优先

平台不再依赖内置 Python bot runtime；外置 ACP bot 和 MCP agent 是同一协议路径。

频道只转发 @

没有中央 coordinator 参与意图判断，路由规则可测试、可审计。

资源写走 Grant

Resource API 读只需频道成员，写需要成员身份和 ACP RBAC Grant。

两类资源分治

Class 1 用 channel_seq 追赶，Class 2 用 memory_files.version 乐观锁。

单实例是本期约束
